分别使用使用Travis与github action 实现hexo自动部署

前置工作

  1. 需要将hexo博客的源码上传到github

Traivs-ci

Travis 包括两个版本

  1. https://travis-ci.com/ 收费使用

  2. https://travis-ci.org/ 针对个人开源项目免费使用

使用Travis

打开第二个免费使用的网站，使用github登录、进入这个页面

https://github.com/settings/installations/11222175

在页面中选择Only select repositories，选中之前上传的hexo博客源码库

之后回到Travis-ci 点击头像，点击 settings按钮

这里选择开始上传的博客仓库

配置环境变量

  • 访问github账户设置 > tokens

  • 生成新的Token Generate new token

  • 填入Token描述，并且选择repo全部权限

  • 将生成的Toke复制保存下来，这个token只出现一次，请妥善保存
    访问Travis CI 的仓库设置、将两个变量填入
